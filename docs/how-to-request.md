# Claude에게 효과적으로 요청하는 법

이 문서는 Claude가 요청을 더 정확하게 수행하기 위한 가이드입니다.

---

## 1. 어디서 작업할지 알려줘

프로젝트에 백엔드(Kotlin)와 프론트엔드(React)가 공존합니다.

**좋은 예:**
> "game 도메인의 Controller를 만들어줘"
> "프론트엔드 game feature에 게임 목록 컴포넌트 만들어줘"

**애매한 예:**
> "게임 목록 만들어줘" → 백엔드 API? 프론트 UI? 둘 다?

## 2. 레이어를 지정해줘 (백엔드)

Hexagonal 구조에서 어느 레이어를 원하는지 말해주면 정확합니다.

| 키워드 | 의미 |
|--------|------|
| "도메인" / "엔티티" | `domain/` — 순수 비즈니스 객체 |
| "유스케이스" / "서비스 로직" | `application/` — 비즈니스 흐름 |
| "포트" / "인터페이스" | `port/` — 계약 정의 |
| "컨트롤러" / "API" | `adapter/in/web/` — REST 엔드포인트 |
| "레포지토리" / "DB" | `adapter/out/persistence/` — DB 구현 |
| "클라이언트" / "외부 API" | `adapter/out/client/` — lichess, chess.com 등 |

**좋은 예:**
> "Game 도메인 엔티티를 만들어줘"
> "lichess 클라이언트 어댑터를 구현해줘"
> "게임 가져오기 유스케이스를 작성해줘"

## 3. 기능 범위를 명확하게

한 번에 전체 기능보다 단계를 나눠서 요청하면 품질이 좋아집니다.

**단계별 요청 (추천):**
> 1. "Game 도메인 엔티티를 만들어줘 (Game, Move, Position)"
> 2. "GameRepository 포트를 정의해줘"
> 3. "R2DBC로 GameRepository를 구현해줘"
> 4. "게임 목록 조회 API를 만들어줘"

**한번에 요청 (가능하지만 리뷰 어려울 수 있음):**
> "lichess에서 게임을 가져와서 DB에 저장하는 전체 흐름을 만들어줘"

## 4. 프론트엔드 요청 시 참고

프론트엔드가 익숙하지 않다면 이렇게 요청해도 됩니다:

**간단하게:**
> "체스보드를 화면에 띄워줘"
> "게임 목록을 보여주는 페이지 만들어줘"

**상세하게 (더 좋은 결과):**
> "game feature에 GameList 컴포넌트를 만들어줘. API에서 게임 목록을 가져와서 테이블로 보여줘"
> "review feature에 ReviewBoard를 만들어줘. chess.js로 수를 탐색하고 react-chessboard로 보드를 보여줘"

모르는 부분은 물어봐 주세요 — 프론트엔드 결정사항이 있으면 선택지를 드립니다.

## 5. 이런 것도 말해주면 좋아

- **"테스트도 같이"** — 테스트 코드를 함께 작성
- **"기존 코드 참고해서"** — 이미 만든 패턴을 따라감
- **"성능 괜찮아?"** — 성능 이슈를 사전에 분석
- **"다른 방법은?"** — 대안을 비교해서 제시
- **"이건 왜 이래?"** — 기존 코드나 구조를 설명

## 6. 요청 템플릿

복사해서 쓰면 됩니다:

```
[대상] 백엔드 / 프론트엔드 / 둘 다
[도메인] game / analysis / review / shared
[레이어] domain / application / port / adapter / 전체 흐름
[내용] 구체적으로 원하는 것
[참고] 추가 맥락이 있으면
```

예시:
```
[대상] 백엔드
[도메인] game
[레이어] 전체 흐름
[내용] lichess API에서 유저네임으로 최근 10게임을 PGN으로 가져와서 DB에 저장
[참고] lichess API는 NDJSON 스트리밍으로 응답함
```
